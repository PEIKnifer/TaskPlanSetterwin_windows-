# 架构与模块说明

## 目标

TaskPlanSetter 旨在提供一个轻量、直观的任务计划管理界面，覆盖“浏览 -> 编辑 -> 高级配置 -> 测试运行”的完整闭环。

## 模块划分

### 1) UI 层

- `Form1.cs` / `Form1.Designer.cs`
  - 主界面交互入口
  - 任务树/任务列表展示
  - 基础编辑与按钮事件（保存、刷新、测试启动、删除、新建）
- `TaskCreateDialog.cs`
  - 新建任务窗口
  - 支持程序路径浏览、默认高级选项
- `TaskAdvancedDialog.cs`
  - 高级选项编辑窗口
  - 读取并回写任务高级配置
- `TechUi.cs`
  - 统一样式与颜色系统
- `TechDialog.cs`
  - 统一提示/确认/错误弹窗

### 2) 业务层

- `Services/TaskPlanService.cs`
  - 封装任务计划读写、删除、立即运行
  - 高级选项模型读写
  - 触发器、Principal、Settings 等配置应用逻辑

## 数据流

1. UI 选择任务（文件夹 + 任务名）
2. UI 调用 `TaskPlanService` 读取模型
3. 用户修改后，UI 调用服务层保存
4. 服务层调用 `TaskScheduler` 库更新任务定义
5. UI 刷新任务列表并重选当前任务

## 关键设计决策

- 使用 `Microsoft.Win32.TaskScheduler` 避免直接操作 COM 细节。
- 默认不强制管理员权限，尽量保证普通用户场景可用。
- 复杂配置集中于“高级”窗口，主界面保持高频操作简洁。
- 提供“测试启动”按钮，降低验证成本（无需重启系统）。

## 异常与权限

- 任务不存在、权限不足、受保护任务写入失败均会明确弹窗。
- 系统任务编辑失败时应提示“以管理员身份运行”。

## 后续可扩展方向

- 任务历史与最后运行结果可视化
- 触发器可视化编辑器（多触发器图形化）
- 导入/导出 XML
- 批量操作（批量启用/禁用/测试）
